Erreurs

Lorsque j'ai codé NBH, j'ai décidé qu'il y avait 2 niveaux d'erreurs : les erreurs fatales (celles qui empêcheraient le bon fonctionnement de NBH), et les autres.

Lors d'une erreur fatale, un fichier "error.log" est créé dans le répertoire où se trouve l'exécutable de NBH. Il contient des informations par rapport à l'erreur. Les différentes erreurs fatales sont détaillées ici !

Les erreurs non fatales sont, quant à elles, répertoriées ici ici et sont décrites dans le fichier "nbh.log" s'il y en a.

Erreurs non fatales

"Il y a trop de XXX pour le personnage YYY."

Cette erreur est dûe au fait qu'il y a trop d'élément XXX dans le fichier d'enregistrement du personnage YYY.

Cela peut venir d'une modification des fichiers personnages par vous-même ; regarder la doc concernant l'enregistrement des fichiers.

"Impossible de charger le fichier de la licence"

Cette erreur ne devrait pas se produire, ell est dûe à une mauvaise inclusion de certains fichiers dans le programme.

Signaler l'erreur sur le site, dans la section "Bug de NBH".

  • "richesses invalides !"

  • "caractéristiques initiales / modifiées invalides !"

    • Regarder si les richesses du personnage correspondent à un enregistrement normal.

Erreurs fatales

NBH ne démarre pas.

Regarder s'il y a un fichier "error.log" / "nbh.log" et lire le message d'erreur dedans..

Remonter la page

"Le fichier de log est inaccessible."

Cette erreur apparaît seulement lorsque NBH ne peut pas accéder au fichier "nbh.log", ce qui normalement, n'arrive pas.

Pour la résoudre, vérifier que NBH peut écrire dans le dossier le contenant.

Remonter la page

"Fichier contenant les origines / métiers inaccessible."

Cette erreur ne devrait pas se produire, ell est dûe à une mauvaise inclusion de certains fichiers dans le programme.

Signaler l'erreur sur le site, dans la section "Bug de NBH".

Remonter la page

"Impossible de créer / ouvrir le fichier de note / groupe / personnage sélectionné."

Cette erreur est normalement impossible, sauf dans le cas où NBH ne pourrai modifier le contenu du dossier dans lequel il se trouve.

Vérifier que NBH peut écrire / lire dans le dossier le contenant.

Cette erreur ne devrait pas arriver en théorie. Je ne sais pas doù elle pourrait provenir, mais, j'en parle quand même.

Elle nous dit que le type d'objet que l'on a voulu attribué est inconnu, et elle donne ce numéro.
